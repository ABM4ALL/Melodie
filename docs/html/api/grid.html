class Melodie.Grid(spot_cls: ClassVar[Spot], scenario=None)ÔÉÅ

Bases: object

Grid is a widely-used discrete space for ABM. Grid contains many Spot`s, each `Spot could contain several agents.

Parameters:

  • spot_cls ‚Äì The class of Spot

  • width ‚Äì The width of Grid

  • height ‚Äì The height of Grid

  • wrap ‚Äì If true, the coordinate overflow will be mapped to another end.

  • caching ‚Äì If true, the neighbors and bound check results will be cached to avoid re-computing.

setup_params(width: int, height: int, wrap=True, caching=True, multi=True)ÔÉÅ

Setup the parameters of the grid.

Parameters:

  • width ‚Äì An integer for the grid width.

  • height ‚Äì An integer for the grid height.

  • wrap ‚Äì A boolean (default True). If True, an agent moving out of the grid on one side will re-enter from the opposite side.

  • caching ‚Äì A boolean (default True). If True, the grid caches the neighbors of each spot to improve performance.

  • multi ‚Äì A boolean (default True). If True, more than one agent can stand on the same spot. If False, an error will be raised when attempting to place multiple agents on one spot.

Returns:

None

setup_agent_locations(category, initial_placement='direct') → NoneÔÉÅ

Add an agent category.

For example, if there are two classes of agents: Wolf(GridAgent) and Sheep(GridAgent), and there are 100 agents with id 0~99 for each class. It is obvious in such a circumstance that we cannot identify an agent only with agent id.So it is essential to use category_name to distinguish two types of agents.

Parameters:

  • category_id ‚Äì The id of new category.

  • category ‚Äì An AgentList object

  • initial_placement ‚Äì A str object stand for initial placement.

Returns:

None

rand_move_agent(agent: GridAgent, category, range_x, range_y)ÔÉÅ

Randomly move an agent with maximum movement range_x in x axis and range_y in y axis.

Parameters:

  • agent ‚Äì Must be Melodie.GridAgent, not Agent. That is because GridAgent has predefined properties required in Grid.

  • range_x ‚Äì The activity range of agent on the x axis.

  • range_y ‚Äì The activity range of agent on the y axis.

For example, if the agent is at (0, 0), range_x=1 and range_y=0, the result can be (-1, 0), (0, 0) or (1, 0). The probability of these three outcomes are equal.

Returns:

(int, int), the new position
